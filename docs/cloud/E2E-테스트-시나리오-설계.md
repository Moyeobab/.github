## 1. 개요

Moyeobab 서비스의 E2E 테스트 시나리오를 설계한다. 테스트 대상 기능을 정리하고, 핵심 플로우를 중심으로 구체적인 테스트 시나리오를 작성한다.

## 2. 핵심 플로우

서비스의 메인 사용자 여정은 다음과 같다:

```
모임 생성 → 초대 코드 공유 → 모임 참여 → 투표 진행 → 결과 조회 → 최종 선택
```

이 플로우에 포함된 기능은 (중요)로 분류하며, E2E 테스트에서 반드시 검증해야 한다.

## 3. 테스트 전제 조건

### 3.1 인증 (Auth)

카카오 로그인은 테스트 대상이 아니라 **테스트 수행을 위한 전제 조건**이다. 카카오 인프라의 동작을 검증하는 것이 아니라, 로그인된 상태에서 서비스 기능이 정상 동작하는지를 확인하는 것이 목적이다.

- 테스트 시작 전 setup 단계에서 로그인 수행
- 인증 상태(`storageState`)를 저장하여 전체 테스트에서 재사용
- 로그아웃, 토큰 갱신은 별도 검증하지 않음

## 4. 기능 목록

### 4.1 온보딩 (Onboarding)

| 기능 | API | 비고 |
|------|-----|------|
| 약관 목록 조회 | `GET /api/v1/onboarding/agreements` | 필수 약관 목록 반환 |
| 약관 전문 조회 | `GET /api/v1/onboarding/agreements/{id}` | 약관 상세 내용 |
| 약관 동의 제출 | `POST /api/v1/onboarding/agreements/consent` | 동의 시 ONBOARDING 상태 전환 |
| 취향 선택지 조회 | `GET /api/v1/onboarding/preferences/choices` | 알레르기, 카테고리 선택지 |
| 취향 저장 | `POST /api/v1/onboarding/preferences` | 저장 시 ACTIVE 상태 전환 |

| 기능 | API | 비고 |

| 내 정보 조회 | `GET /api/v1/member/me` | 프로필 및 취향 정보 |
| 회원 탈퇴 | `DELETE /api/v1/member/me` | DELETED 상태 전환, 카카오 연결 해제 |

### 4.3 모임 (Meeting)

| 기능 | API | 비고 |
|------|-----|------|
| 모임 생성 | `POST /api/v1/meetings` | **(중요)** inviteCode 반환 |
| 내 모임 목록 조회 | `GET /api/v1/meetings` | 커서 기반 페이지네이션 |
| 모임 상세 조회 | `GET /api/v1/meetings/{meetingId}` | **(중요)** 참여자, 투표 상태 포함 |
| 모임 수정 | `PATCH /api/v1/meetings/{meetingId}` | 호스트만 가능 |
| 모임 삭제 | `DELETE /api/v1/meetings/{meetingId}` | 호스트만 가능, soft delete |

| 투표 제출 | `POST /api/v1/meetings/{meetingId}/votes/{voteId}/submissions` | **(중요)** LIKE/DISLIKE/NEUTRAL 일괄 제출 |
| 투표 상태 조회 | `GET /api/v1/meetings/{meetingId}/votes/{voteId}/status` | **(중요)** 제출 현황 확인 |

| 최종 선택 | `POST /api/v1/meetings/{meetingId}/votes/{voteId}/final-selection` | **(중요)** 호스트가 최종 식당 선택 |
| 최종 선택 조회 | `GET /api/v1/meetings/{meetingId}/final-selection` | **(중요)** 선택된 식당 정보 |

## 5. Critical 기능 요약

핵심 플로우(모임 생성 → 초대 → 참여 → 투표 → 결과 → 최종 선택)에 포함된 기능이다. E2E 테스트에서 반드시 검증해야 한다.

| 영역 | 기능 |
|------|------|
| 모임 | 모임 생성, 모임 상세 조회 |
| 참여 | 모임 참여 |
| 투표 | 투표 생성, 후보 조회, 투표 제출, 상태 조회, 결과 조회, 최종 선택, 최종 선택 조회 |

## 6. API-화면 매핑

E2E 테스트는 UI를 통해 API를 호출하므로, 각 기능이 어떤 화면에서 사용되는지 매핑한다.

| 화면 | 관련 API |
|------|----------|
| 약관 동의 | 약관 목록/전문 조회, 약관 동의 제출 |
| 취향 설정 | 취향 선택지 조회, 취향 저장 |
| 홈 (모임 목록) | 내 모임 목록 조회 |
| 모임 생성 | 모임 생성 |
| 모임 상세 | 모임 상세 조회, 투표 상태 조회, 최종 선택 조회 |
| 초대 참여 | 모임 참여 |
| 투표 (스와이프) | 투표 후보 조회, 투표 제출 |
| 투표 결과 | 투표 결과 조회, 최종 선택 |
| 마이페이지 | 내 정보 조회, 회원 탈퇴 |

## 7. 테스트 시나리오

### 7.1 시나리오 목록

| ID | 시나리오 | 범위 | 중요도 |
|----|----------|------|:--------:|
| SC-01 | 핵심 플로우 전체 검증 | 모임 생성 → 참여 → 투표 → 최종 선택 | 중요 |
| SC-02 | 신규 사용자 온보딩 | 약관 동의 → 취향 설정 → 홈 진입 | |
| SC-03 | 모임 목록 및 상세 조회 | 홈에서 모임 목록 확인, 상세 페이지 진입 | |
| SC-04 | 모임 수정 및 삭제 | 호스트가 모임 정보 수정/삭제 | |
| SC-05 | 모임 탈퇴 | 참여자가 모임에서 나가기 | |
| SC-06 | 내 정보 조회 | 마이페이지에서 프로필, 취향 확인 | |

### 7.2 SC-01: 핵심 플로우 전체 검증

서비스의 메인 사용자 여정을 처음부터 끝까지 검증하는 시나리오이다.

**사전 조건**
- 사용자 A (호스트): 로그인 완료, ACTIVE 상태
- 사용자 B (참여자): 로그인 완료, ACTIVE 상태

**테스트 스텝**

| 순서 | 행위자 | 행위 | 기대 결과 |
|:----:|--------|------|-----------|
| 1 | 사용자 A | 모임 생성 페이지에서 모임 정보 입력 후 생성 | 모임 생성 완료, inviteCode 발급 |
| 2 | 사용자 A | 초대 코드 복사 | 클립보드에 초대 코드 복사됨 |
| 3 | 사용자 B | 초대 코드로 모임 참여 | 모임 참여 완료, 모임 상세 페이지 이동 |
| 4 | 사용자 A | 모임 상세에서 투표 시작 | 투표 생성, 후보 식당 목록 로딩 |
| 5 | 사용자 A | 스와이프로 각 후보에 투표 (LIKE/DISLIKE) | 투표 제출 완료 |
| 6 | 사용자 B | 스와이프로 각 후보에 투표 (LIKE/DISLIKE) | 투표 제출 완료 |
| 7 | - | 모든 참여자 투표 완료 확인 | 투표 상태가 COUNTED로 변경 |
| 8 | 사용자 A, B | 투표 결과 페이지 확인 | Top3 식당 목록 표시 |
| 9 | 사용자 A | Top3 중 최종 식당 선택 | 최종 선택 완료 |
| 10 | 사용자 A, B | 모임 상세에서 최종 선택 확인 | 선택된 식당 정보 표시 |

**검증 포인트**
- 모임 생성 시 inviteCode가 정상 발급되는가
- 초대 코드로 다른 사용자가 참여할 수 있는가
- 투표 후보(AI 추천 식당)가 정상 로딩되는가
- 모든 참여자 투표 완료 시 결과가 집계되는가
- 호스트가 최종 선택을 할 수 있는가
- 최종 선택 결과가 모든 참여자에게 표시되는가

### 7.3 SC-02: 신규 사용자 온보딩

신규 가입 후 서비스 이용 전 필수 단계를 검증하는 시나리오이다.

**사전 조건**
- 신규 사용자: 카카오 로그인 완료, PENDING 상태 (약관 미동의)

**테스트 스텝**

| 순서 | 행위 | 기대 결과 |
|:----:|------|-----------|
| 1 | 약관 동의 페이지 진입 | 필수 약관 목록 표시 |
| 2 | 약관 전문 보기 클릭 | 약관 상세 내용 표시 |
| 3 | 필수 약관 전체 동의 후 제출 | ONBOARDING 상태로 전환, 취향 설정 페이지 이동 |
| 4 | 취향 선택지 확인 | 알레르기, 선호/비선호 카테고리 선택지 표시 |
| 5 | 취향 선택 후 저장 | ACTIVE 상태로 전환, 홈 페이지 이동 |

**검증 포인트**
- 약관 미동의 시 서비스 이용이 차단되는가
- 약관 동의 후 취향 설정으로 자동 이동하는가
- 취향 저장 후 홈에 정상 진입하는가

### 7.4 SC-03: 모임 목록 및 상세 조회

홈 화면과 모임 상세 화면의 정상 동작을 검증하는 시나리오이다.

**사전 조건**
- 사용자: 로그인 완료, ACTIVE 상태
- 참여 중인 모임이 1개 이상 존재

**테스트 스텝**

| 순서 | 행위 | 기대 결과 |
|:----:|------|-----------|
| 1 | 홈 페이지 진입 | 참여 중인 모임 목록 표시 |
| 2 | 모임 카드 클릭 | 모임 상세 페이지 이동 |
| 3 | 모임 상세 정보 확인 | 제목, 일시, 장소, 참여자, 투표 상태 표시 |

**검증 포인트**
- 모임 목록이 정상 로딩되는가
- 페이지네이션(커서 기반)이 동작하는가
- 모임 상세 정보가 정확한가

### 7.5 SC-04: 모임 수정 및 삭제

호스트의 모임 관리 기능을 검증하는 시나리오이다.

**사전 조건**
- 사용자: 로그인 완료, 모임의 호스트

**테스트 스텝**

| 순서 | 행위 | 기대 결과 |
|:----:|------|-----------|
| 1 | 모임 상세에서 수정 버튼 클릭 | 모임 수정 페이지 이동 |
| 2 | 모임 제목 변경 후 저장 | 변경된 제목이 반영됨 |
| 3 | 모임 상세에서 삭제 버튼 클릭 | 삭제 확인 모달 표시 |
| 4 | 삭제 확인 | 모임 삭제 완료, 홈으로 이동 |

**검증 포인트**
- 호스트만 수정/삭제 버튼이 표시되는가
- 수정 내용이 정상 반영되는가
- 삭제 후 모임 목록에서 제거되는가

### 7.6 SC-05: 모임 탈퇴

참여자의 모임 탈퇴 기능을 검증하는 시나리오이다.

**사전 조건**
- 사용자: 로그인 완료, 모임의 참여자 (호스트 아님)

**테스트 스텝**

| 순서 | 행위 | 기대 결과 |
|:----:|------|-----------|
| 1 | 모임 상세에서 탈퇴 버튼 클릭 | 탈퇴 확인 모달 표시 |
| 2 | 탈퇴 확인 | 모임 탈퇴 완료, 홈으로 이동 |
| 3 | 홈에서 모임 목록 확인 | 탈퇴한 모임이 목록에서 제거됨 |

**검증 포인트**
- 호스트는 탈퇴 버튼이 표시되지 않는가
- 탈퇴 후 모임 목록에서 제거되는가

### 7.7 SC-06: 내 정보 조회

마이페이지 기능을 검증하는 시나리오이다.

**사전 조건**
- 사용자: 로그인 완료, ACTIVE 상태

**테스트 스텝**

| 순서 | 행위 | 기대 결과 |
|:----:|------|-----------|
| 1 | 마이페이지 진입 | 프로필 정보 표시 (닉네임, 프로필 이미지) |
| 2 | 취향 정보 확인 | 설정한 알레르기, 선호/비선호 카테고리 표시 |

**검증 포인트**
- 카카오에서 가져온 프로필 정보가 정확한가
- 온보딩에서 설정한 취향 정보가 정확한가
