## 1. 주요 부하 테스트 도구 기술 분석

### 1.1 k6 (by Grafana Labs)

* **엔진 및 언어**: Go 언어로 작성된 런타임 위에서 **JavaScript(ES6)** 스크립트를 실행합니다.
* **리소스 효율성**: 단일 프로세스 내에서 가상 사용자(VU)를 고성능 **Go 루틴**으로 처리하여, JVM 기반 도구 대비 메모리 점유율이 낮습니다.
* **CI/CD 환경**: CLI 기반의 단일 바이너리로 동작하며, **Thresholds** 기능을 통해 에러율이나 응답 시간 기준에 따른 자동 Pass/Fail 판정이 가능합니다.
* **확장성**: HTTP/1.1, HTTP/2, WebSocket, gRPC를 네이티브로 지원하며, Grafana 및 Prometheus 연동을 통한 실시간 시각화를 지원합니다.
* **분산 테스트**: 오픈소스 버전은 단일 노드 실행이 기본이며, 대규모 환경에서는 k6 Operator(k8s)를 활용합니다.

### 1.2 Apache JMeter

* **엔진 및 언어**: **Java(JVM)** 기반의 스레드(Thread) 모델을 사용하며, XML 형식으로 테스트 플랜을 저장합니다.
* **인프라 요구사항**: 가상 사용자마다 개별 스레드를 할당하므로, 대규모 부하 생성 시 CPU와 메모리 리소스 소모가 상대적으로 큽니다.
* **지원 범위**: 20년 이상의 역사를 바탕으로 JDBC, FTP, SMTP 등 광범위한 네트워크 프로토콜을 플러그인 형태로 지원합니다.
* **운용 방식**: GUI 모드에서 시나리오를 설계하고, 실제 테스트는 리소스 절약을 위해 Non-GUI(CLI) 모드에서 실행하는 것을 권장합니다.

### 1.3 Gatling

* **엔진 및 언어**: JVM 위에서 **Akka 툴킷과 Netty**를 사용한 비동기 이벤트 기반 모델로 동작합니다. Scala, Java, Kotlin DSL을 지원합니다.
* **성능 특성**: 비동기 I/O를 사용하여 스레드 차단 없이 적은 자원으로도 높은 동시성을 처리합니다.
* **결과 분석**: 테스트 종료 후 별도의 도구 연동 없이도 각 요청별 분포와 통계가 포함된 상세한 **자체 HTML 리포트**를 생성합니다.
* **학습 곡선**: 코드 기반 시나리오 작성을 지향하며, Scala 기반의 함수형 프로그래밍 개념이 스크립트에 반영되어 있습니다.

### 1.4 Locust

* **엔진 및 언어**: Python 언어 기반의 **이벤트 루프(gevent)** 모델을 채택하고 있습니다.
* **분산 처리**: Master-Worker 구조를 기본 지원하여 여러 대의 서버에서 부하를 분산 생성하고 중앙에서 취합하기 용이합니다.
* **런타임 효율**: Python의 GIL 제약으로 단일 워커 성능은 제한적일 수 있으나, 코어당 워커를 늘리는 방식으로 확장 가능합니다.
* **모니터링**: 테스트 진행 중 웹 대시보드를 통해 실시간 요청 수와 응답 시간 그래프를 확인할 수 있습니다.

## 2. Moyeobab의 k6 선정 이유

### 2.1 스파이크 트래픽(Spike Traffic) 대응에 최적화

Moyeobab은 점심/저녁 시간대에 트래픽이 급격히 몰리는 '스파이크 패턴'이 서비스의 핵심입니다. k6는 `stages` 옵션을 통해 부하의 증감(Ramp-up/down)을 코드로 정교하게 제어할 수 있습니다.

* **기술적 이점**: GUI 설정 방식보다 부하 패턴의 가독성이 높으며, 특정 시간대의 급격한 트래픽 변화를 선언적으로 관리하기에 용이합니다.

### 2.2 단일 인스턴스 기반의 고효율·저비용 운용

MAU 500 규모의 소규모 서비스에서 테스트를 위해 대규모 인프라를 별도로 구축하는 것은 비효율적입니다. k6는 Go 엔진의 효율성을 바탕으로 저사양 인스턴스에서도 수천 명의 가상 사용자(VU)를 시뮬레이션할 수 있습니다.

* **기술적 이점**: JVM 기반 도구와 달리 적은 리소스로 부하를 생성할 수 있어 로컬 환경이나 단일 CI 서버만으로 충분한 테스트가 가능합니다.

### 2.3 GitHub Actions 기반의 자동화 편의성

6명의 개발자가 협업하며 GitHub Actions를 통해 CI/CD를 수행하는 환경에 k6는 가장 적합한 도구입니다.

* **기술적 이점**: CLI 중심의 단일 바이너리 도구로 워크플로우 연동이 간편하며, `Thresholds` 기능을 활용해 성능 저하 시 배포를 자동 중단하는 **성능 게이트(Performance Gate)** 구축이 쉽습니다.

### 2.4 시나리오 분리 및 복합 테스트 지원

모임 참여, 투표 제출, 결과 조회 등 구간별로 성능 요구사항이 다른 Moyeobab의 특성을 잘 반영할 수 있습니다.

* **기술적 이점**: 하나의 스크립트 내에서 각 API별로 서로 다른 부하 정책과 지연 시간(Latency) 임계치를 설정할 수 있어, 병목 지점을 정밀하게 파악하기에 유리합니다.
