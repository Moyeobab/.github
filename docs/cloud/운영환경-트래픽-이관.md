<h2>배경</h2>
<p>v2 인프라 구축 완료 후 prod 환경으로 트래픽을 전환. 서비스 안정성을 고려하여 한 번에 전환하지 않고 단계적 카나리 방식으로 트래픽을 이관. 이상 징후 발생 시 즉시 롤백 가능한 구조로 진행.</p>
<h2>작업 내용</h2>
<h3>트래픽 이관 전략</h3>
<p>초기 계획은 Route 53 가중치 기반 라우팅으로 v1 Nginx EC2와 v2 ALB를 분리하는 방식이었으나, dev 환경에서의 트래픽 이관 사례를 확인하며 ALB 방식이 더 효과적이라 판단하여 채택.</p>
<p><strong>Route 53 방식 대비 ALB 대상 그룹 방식이 무중단 이관에 적합한 이유</strong></p>
<ul>
<li>DNS TTL 없이 가중치 변경이 즉시 반영되어 이상 징후 발생 시 롤백 속도가 빠름</li>
<li>Route 53 방식은 롤백해도 DNS TTL 동안 일부 사용자가 계속 v2로 접근할 수 있음</li>
<li>단일 진입점(<code>api.moyeobab.com → v2 prod ALB</code>)에서 v1/v2 트래픽을 통합 관리 가능</li>
</ul>
